# ayx-site-journal

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可能包含静态内容、信息展示或功能入口，适用于快速浏览与分发。

仓库本身不针对任何特定域名或网站，仅作为一个通用的页面存储与版本管理空间。

## 目录说明

```
.
├── pages/          # 存放所有独立 HTML 页面文件
├── assets/         # 页面引用的静态资源（如样式、脚本、图片等）
├── index.html      # 仓库首页（可选，用于导航或说明）
└── README.md       # 本文件
```

- **pages/**：每个 HTML 文件为一个独立页面，按需命名。
- **assets/**：根据具体页面需求存放资源文件，建议按页面或功能模块组织子目录。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，可脱离本仓库直接运行。
- 仓库中的页面可能包含实验性内容、临时归档或长期维护项目。
- 页面之间的关联性不强，请根据实际需要查看或使用。

## 维护说明

- 提交新页面时，请将 HTML 文件放入 `pages/` 目录，并确保相关资源同步上传至 `assets/` 目录。
- 建议在提交信息中简要描述页面的用途或变更内容。
- 如页面不再需要，可将其移至 `archive/` 目录（如创建）或直接删除，并更新相关引用。
- 欢迎通过 Issue 或 Pull Request 提出建议、报告问题或贡献新页面。
